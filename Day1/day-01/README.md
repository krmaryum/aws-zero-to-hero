# Day 1 Completed – 7 Days of AWS Challenge

Today I completed **Day 1** of the **7 Days of AWS Challenge**.

The focus of Day 1 was understanding the basics of cloud computing and AWS. I learned that cloud computing allows users to access IT resources such as servers, storage, databases, networking, and security over the internet without buying physical hardware.

---

## Key Topics I Covered Today

### 1. AWS Pricing Models

I learned about:

- Pay-as-you-go
- Reserved Instances
- Spot Instances
- Free Tier
- Savings Plans

My main takeaway is that **Pay-as-you-go** is very useful for beginners and startups because it gives flexibility without long-term commitment.

However, billing should always be monitored carefully.

---

### 2. Cloud Models

I learned the difference between:

- On-premises
- Cloud
- Hybrid

Simple understanding:

| Model | Meaning |
|---|---|
| On-premises | Gives full control |
| Cloud | Provides speed and scalability |
| Hybrid | Gives a balance between control and flexibility |

---

### 3. Service Models

I explored:

- IaaS
- PaaS
- SaaS

Quick examples:

| Service Model | Meaning | AWS Example |
|---|---|---|
| IaaS | Infrastructure as a Service | EC2 |
| PaaS | Platform as a Service | Lambda |
| SaaS | Software as a Service | WorkMail |

---

### 4. AWS History and Key Milestones

I learned how AWS started in **2006** and how services like **S3**, **EC2**, and **Lambda** helped change the way companies use infrastructure.

AWS made it possible for companies, startups, developers, and learners to use cloud services on demand instead of buying and maintaining physical servers.

---

## My Simple Understanding from Day 1

Cloud computing is not only about technology. It is also about:

- Flexibility
- Cost management
- Scalability
- Security
- Better infrastructure decisions
